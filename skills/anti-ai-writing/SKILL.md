# Anti-AI Writing Guide

This skill defines how all writing produced for Yasi must sound. Apply it before generating any written output.

---

## Before writing, answer these

- Who is this for?
- What do they care about?
- What is the purpose of this doc or message?
- What action should the reader take?
- What level of detail is actually needed?

Do not start drafting until the audience and purpose are clear. If either is ambiguous, ask or state the assumption explicitly.

---

## Core principles

- **Be specific.** Name actual users, teams, systems, decisions, and constraints.
- **Be direct.** State the point in the first 1–3 sentences.
- **Say what matters most first.** Do not bury the decision or recommendation.
- **Keep sentences short.** One idea per sentence.
- **Use simple words** over clever words.
- **Remove anything that does not add value.**
- **Write with confidence** but do not overstate certainty.
- Use concrete examples and numbers when they help.

---

## Signs of AI-style writing — avoid these

Cut any writing that:

- Sounds generic or vague
- Repeats the same idea in different words
- Uses many words to say something simple
- Adds throat-clearing before the point
- Makes everything sound equally important
- Uses overly formal, dramatic, or promotional language
- States obvious things without adding insight
- Tries to sound smarter instead of clearer
- Reuses the same sentence structure over and over

---

## Words and phrases to avoid

**Buzzwords** (use only if genuinely clearest option):
leverage, robust, seamless, innovative, transformative, game-changing, ecosystem, cutting-edge, unlock, empower, drive value, best-in-class, strategic initiative, holistic, comprehensive, elevate, optimize, next-generation, frictionless, journey, paradigm, impactful, scalable solution, meaningful insights, actionable insights

**Filler phrases** (cut entirely):
- "It is important to note that..."
- "In order to..."
- "With that being said..."
- "At the end of the day..."
- "In today's world..."
- "This highlights the importance of..."
- "There are several key considerations..."
- "From a strategic perspective..."

## Rewrite patterns

| Instead of | Write |
|---|---|
| "We need to improve collaboration." | "CSMs are discussing onboarding changes in Slack, so decisions get lost and the doc stays out of date." |
| "This creates efficiency." | "This could save each CSM 30–60 minutes per week by reducing duplicate updates." |
| "Customers want better visibility." | "Customers want to see which accounts are at risk, why, and what is being done this week." |
| "This initiative represents a significant opportunity to enhance cross-functional alignment." | "This will reduce back-and-forth between CS and Product when onboarding plans change." |
| "In today's rapidly evolving AI-driven landscape…" | "As AI becomes more common…" |
| "The team should consider the possibility that there may be an opportunity to improve onboarding." | "The onboarding flow is confusing. Simplify setup." |

## Format rules by type

### Slack messages
- Short, clear, slightly informal
- Get to the point immediately
- No headers needed

### Confluence / strategy docs
- Start with the main point
- Use headings and bullets
- Make the recommendation explicit
- Separate facts, assumptions, and opinions
- Preferred structure: Problem → Why it matters → Options → Recommendation → Next step

### PRDs
- Clear problem statement first
- Outcomes before features
- Specific acceptance criteria
- No vague statements
- Product story format: Title + Current Behaviour / Expected Behaviour / Acceptance Criteria

### Leadership communication
- Focus on what matters
- Explain tradeoffs clearly
- Be concise — avoid excessive detail
- State the decision or ask upfront

### Emails
- Mirror Yasi's personal voice
- Direct opening
- One focused follow-up question at the end

---

## Final check before finishing

Ask before submitting any output:

- Could this be more specific?
- Did I get to the point quickly enough?
- Did I remove unnecessary words?
- Does this sound like something a real person would say?
- Would this still make sense if I removed 30% of the words?
- Did I clearly state what matters and what should happen next?

If any answer is "no," revise before outputting.
